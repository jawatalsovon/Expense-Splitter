Project: Build a full-stack expense splitting web app — a lightweight Splitwise alternative. It must be publicly usable by anyone, with persistent cloud data, authentication, and real-time sync.

TECH STACK

Frontend: React, Tailwind CSS, shadcn/ui components
Backend/DB/Auth/Realtime: Supabase
Routing: React Router v6
State: React Context or Zustand
Hosting target: Vercel (just build for production export)


AUTHENTICATION

Email + password signup and login using Supabase Auth
Optional: Google OAuth login button
After signup, prompt user to set a display name (stored in a profiles table)
Protected routes — unauthenticated users are redirected to login
Persistent session (stay logged in on refresh)
Simple, clean auth screens — no clutter

BALANCE CALCULATION LOGIC (implement carefully)
For each expense, a member's net contribution is:
net = amount_paid - share_amount
If net > 0 → others owe them
If net < 0 → they owe others
Aggregate across all expenses in a group to get each member's total net balance.
Debt simplification algorithm:

Calculate net balance for each member
Separate into creditors (positive) and debtors (negative)
Use a greedy matching: match the largest debtor to the largest creditor, settle as much as possible, repeat until all balances are zero
This produces the minimum number of transactions needed

This logic should run client-side after fetching all expense data for the group. Also factor in settled payments from the settlements table.

REAL-TIME

Use Supabase Realtime subscriptions on the expenses, expense_payers, expense_splits, and settlements tables scoped to the current group
When any group member adds an expense or settles a debt, all other members on the Group Page see it update instantly without refresh
Show a subtle toast notification: "Tasin added an expense: Dinner ৳350"


UI/UX REQUIREMENTS

Mobile-first, but functional on desktop too (max-width container centered on wide screens)
Each user has a persistent avatar color (from a palette of 8 distinct colors assigned at signup) — used as a colored circle avatar everywhere instead of a photo
Currency: ৳ (Bangladeshi Taka) — but make the currency symbol a config constant so it can be changed
Green for positive balances, red for negative, gray for zero
Smooth transitions and loading skeletons (not blank screens while fetching)
Empty states with helpful text (e.g. "No expenses yet — add the first one!")
All destructive actions (delete expense, settle up, leave group) require a confirmation dialog
Toast notifications for all key actions (expense added, settled, error, etc.)
Responsive bottom sheets on mobile for the Add Expense flow


EDGE CASES TO HANDLE

A payer can also be in the split (they paid but also owe a share — their net adjusts correctly)
An expense where only one person both pays and splits = no debt created
If a user leaves a group (future-proofing): their historical expenses remain, balances still calculate correctly
Prevent duplicate group joins via invite code
Validate that expense amounts are positive numbers only
If the sum of payer amounts ≠ total expense amount, block submission with a clear error
